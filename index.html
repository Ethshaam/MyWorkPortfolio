Storage

Amazon Block Storage (ABS) is a storage system which AWS provides. ABS comes added onto your EC2 instance. There are many different storage types you can have within AWS they are; ABS, EBS (elastic block storage) but this service can only be mounted if the ec2 instance has been launched within the same Availability zone. An availability zone is a logical data center in a region. The main difference between S3 and EBS is that S3 is a persistant storage device and can be accessed from anywhere unlike EBS.

EBS is an block storage system that is best used for storing persistant data. As mentioned earlier it is a highly available storage for use with an ec2 AMI

What is Persistant Storag?
Persistant storage is any data storage that retains data after the power to the device is off. Can also be referred to as Non volatile Storage
Each EBS volume is automatically replicated to protect the user from component failure which offers the user high availability and durability. EBS is also consistent and runs low-latency performans
The user also has the ability to scale up and down within minutes,

Block Storage vs Object Storage Block Storage you change only the block which contains the character. Object Storage is you have to change to whole file which requires updating. The difference has a major impact on the throughput, latency and cost of the storage solution

You can use ABS for the following uses:
Boot volumes and storage for EC2 instances
Data storage file system
Database hosts
Enterprise applications

Cloud Computing / EC2

Amazon EC2 stands for: Amazon Elastic Computer Cloud. EC2 is a web service which is provided by Amazon Web Services it is designed to make web-scale cloud computing easier for developers. EC2s interface allows the user to obtain and configure capacity with minimum friction which means it isn't very hard to do. It provides the user with free range control over the computing resources which Amazon Provides you with.

EC2 allows you to build an instance within the AWS environment. An instance is a Virtual Machine which gives you free range of different operation systems from Mac OS, Windows OS and Linux OS. It does this by giving the user a choice of: Processor, Storage, Networking, Operating System and purchase Model.

Functionality of EC2

EC2 allows you to choose a pre-configured, AMI Amazon Machine Image to get you up and running immediately. Or you can create an AMI which contains the applications, library, data and configured settings

Configure the networking and security of your instance

Choose which instance type you would like to work on, with this you are also able to terminate, monitor and start as many instances as you would like

You have free range as to where you can set up your instance wether that be in England or in the United States

Pay only for what you use such as the data that the instance consumes. This is normally on an hourly basis


How would you use EC2?


As mentioned above EC2 is primarly used to create new instances (VMs) and to do this you would need to:

  1. Go to the amazon web management console and go to services and click EC2
  2. An AMI is an Image which is required to launch an Instance.
  3. Then configure the Operation system or an application server with applications
  4. Note: Each instance type has its uses. The user is able to choose: CPU, MEMORY, STORAGE and Networking capacity and gives you the flexibility to choose the mix that is required
  5. Configuring instance details

    The Instance details are things what network you would like to have as you are able to have ones for different use cases. You are able to have ones for: Development, Testing and production

    Here you can also safe guard yourself from accidental termination. When instance in terminated the resources are released, which means it cannot be re-started.

  6. Add Storage
    The storage which the instance has is from a virtual disk known as Amazon Block Storage, here you are able to tell the instance how much memory you want it to have

  7. Add Tags
    The tags in the instance gives it identification and recognition. It has 2 key pairs (Simialr to a dictionairy in other coding languages)

  8. Configure security Group
    A security group acts as a firewall that controls traffic for one or more instances. NOTE: A firewall only monitors traffic coming into the instance The security group will then have a group name and a description

  9. Review and Launch Instance You can know launch the instance
